Dune: Awakening Character Creation Guide

Character pages age quickly when they turn old screenshots, class lists, or community assumptions into fixed rules. Start with the current official source, then use a clear boundary between confirmed information and personal experimentation.

1. Ask What the Source Actually Confirms

An official page, trailer, or store description can establish visible framing and explicitly named systems. It cannot prove every option, progression value, dialogue effect, or long-term build result that a reader expects a guide to answer.

  • Quote the source only where it is specific.
  • Keep screenshots as visual context, not a complete option list.
  • Record the source date beside a detailed claim.

2. Separate Appearance, Story, and Mechanics

Do not assume a visual choice changes a combat, progression, or dialogue outcome unless a current primary source says so. A useful guide explains what a player should verify before committing to a choice and what remains a personal preference.

  • Mark cosmetic choices as cosmetic unless confirmed otherwise.
  • Label story interpretation as interpretation.
  • Keep a build claim version-aware.

3. Test One Decision at a Time

If you publish a personal test, name the platform, version, setting, and result. Change one relevant choice rather than several, and do not elevate an anecdote into a universal recommendation without corroborating evidence.

  • State the question being tested.
  • Describe only the observed result.
  • Link the official update if it changes the context.

4. Build a Better First Session

For a new player, preparation is more valuable than a speculative perfect build. Start with a small survival objective, decide what supplies support it, and use the beginner survival guide and base planning guide to plan the first return loop.
